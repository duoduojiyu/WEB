# AI 学习指南

帮助用户高效学习AI技能、掌握新知识的系统性方法。

## 快速开始

当用户提到想学习某个AI技能或需要学习方法时，按照以下流程：

1. **评估现状**：询问用户当前水平、目标、可用时间
2. **制定计划**：根据评估提供分阶段学习路线
3. **推荐资源**：给出具体的学习资源和工具
4. **实践建议**：提供配套练习项目

## 学习流程框架

### 阶段一：明确目标

询问用户以下问题（选择最相关的）：
- 你的学习目标是什么？（工作应用/兴趣探索/职业发展）
- 每周可投入多少时间学习？
- 有相关基础吗？（编程/数学/其他AI经验）

### 阶段二：制定学习路线

根据目标提供3-4个阶段的学习计划：

| 学习方向 | 阶段1 | 阶段2 | 阶段3 | 阶段4 |
|---------|-------|-------|-------|-------|
| 提示词工程 | 基础概念 | 结构化提示 | 高级技巧 | 项目实战 |
| AI编程辅助 | 工具使用 | 高效协作 | 代码生成 | 项目开发 |
| AI应用开发 | API基础 | 框架学习 | 应用构建 | 部署优化 |
| ML/DL入门 | Python基础 | 机器学习 | 深度学习 | 项目实践 |

### 阶段三：知识整理方法

推荐用户采用以下方式整理知识：

**笔记结构模板：**
```
## 知识点名称
- 核心概念：一句话概括
- 关键要点：3-5个要点
- 代码/示例：实际可运行的例子
- 常见误区：容易犯的错误
- 关联知识：相关概念链接
```

**复习周期建议：**
- 当天：学习后立即回顾
- 第2天：快速复习
- 第7天：深度复习
- 第30天：应用检验

### 阶段四：实践项目建议

为每个学习阶段匹配对应难度的项目：

| 难度 | 提示词工程 | AI编程 | AI应用 | ML/DL |
|-----|-----------|--------|--------|-------|
| 初级 | 个人助手提示优化 | 代码重构辅助 | 简单聊天机器人 | 数据可视化 |
| 中级 | 自动化工作流 | 完整项目开发 | API集成应用 | 分类/回归模型 |
| 高级 | 多智能体系统 | AI驱动工具 | 复杂AI产品 | 深度学习模型 |

## 知识点配套练习

每个知识点学习后，必须配合练习题巩固。根据知识类型自动生成相应练习。

### 练习设计原则

1. **即时性**：学习完立即练习，不要拖延
2. **多样性**：概念理解 + 应用实践 + 问题排查
3. **递进性**：从简单回忆到综合应用
4. **真实性**：尽量用实际场景的案例
5. **可验证**：练习结果可以自我检查

### 知识类型与练习类型对应

| 知识类型 | 配套练习 | 数量建议 |
|---------|---------|---------|
| 概念/定义 | 概念辨析题 + 举例题 | 2-3题 |
| 原理/机制 | 原理分析题 + 画图题 | 2-3题 |
| API/代码 | 代码补全 + 错误排查 + 功能实现 | 3-5题 |
| 算法/模型 | 手推公式 + 代码实现 + 调参实验 | 3-4题 |
| 工具/框架 | 配置部署 + 实际项目应用 | 2-3题 |
| 最佳实践 | 场景分析题 + 方案设计题 | 2题 |

### 练习生成方法

当用户学习完一个知识点后，按照以下流程生成练习：

1. **识别知识类型**：判断是概念/代码/算法/工具
2. **选择练习模板**：根据类型选择2-3个模板
3. **设计具体题目**：结合知识点设计题目
4. **提供参考答案**：给出详细的答案和解析
5. **说明验证方法**：告诉用户如何检查答案

## 常见学习陷阱提醒

1. **只看不练**：每个知识点必须有实践
2. **贪多求全**：一次只聚焦一个技能
3. **忽视基础**：基础概念要扎实
4. **缺乏复习**：按周期回顾已学内容
5. **孤军奋战**：多参与社区交流

## 使用示例

**用户说**："我想学习提示词工程"

**响应流程**：
1. 询问：学习目的？每周时间？现有基础？
2. 给出4阶段学习计划（每阶段1-2周）

**用户说**："我怎么记不住这些知识"

**响应流程**：
1. 了解：具体是什么内容记不住？怎么学的？

3. 提供参考答案和解析
4. 建议记录练习完成情况
